# Coordination-Intelligence-Layer-POC-Details

Welcome to the Coordination Intelligence Layer (CIL) Proof of Concept repository.

The Coordination Intelligence Layer is a robust, domain-neutral orchestration and coordination middleware. It evaluates how localized events or failures ripple through complex interdependent systems—ranging from Building Information Modeling (BIM) architectures to distributed microservices.

## 📖 Official Documentation

**All technical documentation, architectural decisions, and enterprise-level manuals have been meticulously organized and can be found in the `coordination-intelligence-layer-docs` folder.**

👉 **[View the Complete Documentation Here](coordination-intelligence-layer-docs/README.md)**

Inside the documentation repository you will find:
- **Core Concepts:** Understanding domain-neutral propagation and coordination.
- **Architecture:** High-level topology, component architecture, and the dual Go/Python propagation boundary.
- **Engines:** Deep dives into the Graph, Propagation, Decision, and Simulation engines.
- **Explainability:** Details on CIL's transparent reasoning chains, confidence scoring, and causal narratives (v0.2).
- **Adapters:** How CIL interacts with external systems like BIM pipelines.
- **Architectural Decision Records (ADRs):** The historical context explaining why specific engineering choices were made.

### Directory Structure Overview

- `/CIL-Details/` - Contains raw reference materials and unformatted notes for the POC.
- `/coordination-intelligence-layer-docs/` - **The official, structured enterprise documentation.**

For instructions on getting started, running the CLI, or navigating the APIs, please dive straight into the official documentation folder!
